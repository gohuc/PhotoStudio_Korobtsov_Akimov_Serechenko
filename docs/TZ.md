Техническое задание
Веб-сайт фотостудии с онлайн-бронированием и админ-панелью
1. Общая информация
Цель проекта:
Разработка масштабируемого веб-сайта фотостудии с интерактивной системой онлайн-бронирования, выбором пакетов и дополнительных услуг, автоматическим расчётом итоговой стоимости и полноценной административной панелью для управления контентом и бронированиями.
Тип проекта:
Веб-приложение (responsive), доступное с компьютеров и мобильных устройств.
Референс по логике и визуальной подаче:
https://www.maiko-maiko.com
(используется как ориентир по структуре, UX и подаче, без копирования дизайна).
________________________________________
2. Структура сайта (публичная часть)
Сайт должен содержать следующие основные разделы (вкладки):
1.	Главная
o	Общее описание студии
o	Преимущества
o	Краткое описание пакетов
o	Призыв к бронированию
o	Фотографии / визуальные блоки
2.	Бронь
o	Интерактивный календарь
o	Выбор даты и времени
o	Выбор пакета услуг
o	Выбор дополнительных услуг
o	Автоматический расчёт итоговой стоимости
o	Форма ввода данных клиента
3.	Галерея
o	Фотографии работ
o	Разделение по образам / категориям 
o	Все изображения добавляются через админ-панель
4.	О нас
o	Информация о студии
o	Концепция, миссия, формат работы
o	Контент редактируется через админ-панель
5.	Пакеты услуг
o	Подробное описание пакета «всё включено»
o	Отдельный блок под каждый образ
o	Описание: что входит, количество фото, длительность, цена
o	Контент полностью управляется из админ-панели
________________________________________
3. Функциональные требования (бронирование)
3.1 Интерактивный календарь
•	Отображение доступных дат и временных слотов
•	Недоступные слоты визуально заблокированы
•	Администратор может:
o	закрывать конкретные даты
o	закрывать конкретное время
o	вручную блокировать слоты (без брони клиента)
3.2 Типы бронирования (опции)
Пользователь может выбрать один из вариантов:
1.	Бронь локации
2.	Образ + локация
3.	Фотограф + локация
4.	Пакет «Всё включено»
o	фотограф
o	локация
o	образ
Все варианты:
•	настраиваются администратором
•	могут иметь разную цену
•	могут быть включены / выключены через админ-панель
________________________________________
4. Дополнительные услуги
4.1 Список дополнительных услуг (пример)
•	Дым-машина
•	Групповая фотосессия
•	Дополнительный свет
•	Дополнительные декорации
•	Макияж
•	Причёска
•	Дополнительные фотографии
•	Другие услуги (возможность добавления в будущем)
4.2 Требования
•	Все доп. услуги:
o	добавляются / редактируются / удаляются через админ-панель
o	имеют цену
o	могут быть включены или отключены
•	Администратор может:
o	группировать услуги по заголовкам (например: «Свет», «Образы», «Комфорт»)
o	менять порядок отображения
•	Клиент может выбрать несколько дополнительных услуг
•	Итоговая стоимость пересчитывается автоматически в реальном времени
________________________________________
5. Расчёт стоимости
Система должна:
•	учитывать:
o	выбранный тип бронирования
o	выбранный пакет
o	выбранные дополнительные услуги
•	отображать:
o	детализированный список выбранных опций
o	итоговую сумму
•	пересчитывать стоимость без перезагрузки страницы
________________________________________
6. Административная панель
6.1 Общие требования
•	Веб-админка
•	Доступ только для администратора
•	Масштабируемая архитектура (возможность добавления новых разделов и логики)
6.2 Управление контентом
Администратор может:
•	добавлять / редактировать / удалять:
o	страницы сайта
o	тексты
o	изображения
o	пакеты услуг
o	дополнительные услуги
o	заголовки для группировки услуг
•	управлять галереей:
o	загрузка фото
o	удаление фото
o	изменение порядка
6.3 Управление бронированиями
Администратор видит:
•	список всех бронирований
•	по каждому бронированию:
o	имя клиента
o	контакты (email / телефон)
o	дата и время
o	выбранный пакет
o	все дополнительные услуги
o	итоговая сумма
Администратор может:
•	подтверждать бронирование через админ-панель
•	отменять бронирование
•	блокировать слот вручную (без клиента)
________________________________________
7. Уведомления
7.1 Клиенту
После подтверждения бронирования администратором:
•	клиенту автоматически отправляется email
•	в письме:
o	подтверждение брони
o	дата и время
o	выбранные услуги
o	итоговая сумма
o	контактная информация студии
________________________________________
8. Адаптивность и устройства
•	Сайт должен корректно отображаться:
o	на компьютерах
o	на планшетах
o	на мобильных устройствах
•	Mobile-first или полноценная адаптивная верстка
•	Удобное бронирование с телефона
________________________________________
9. Масштабируемость системы
Архитектура должна предусматривать возможность:
•	добавления новых услуг
•	добавления новых пакетов
•	добавления новых типов бронирования
•	подключения онлайн-оплаты в будущем
•	расширения логики (акции, промокоды, личный кабинет клиента и т.п.)
________________________________________
10. Итог
Результатом работы должен быть:
•	полностью функциональный сайт фотостудии
•	с интерактивной системой бронирования
•	гибкой админ-панелью
•	автоматическим расчётом стоимости
•	email-уведомлениями
•	адаптивным интерфейсом
•	готовый к масштабированию проект
